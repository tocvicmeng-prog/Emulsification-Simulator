# Audit Report: Module 2 v5.9 Comprehensive Plan

Date: 2026-04-12  
Auditor role: computational simulation scientist, wet-lab chemistry researcher, physicochemical model reviewer  
Scope: `docs/28_v59_plan.md` and the current EmulSim M2/M3 implementation.

## 1. Executive Verdict

The v5.9 plan addresses the correct unresolved scientific and system-integration gaps: IMAC metal charging, M2-to-M3 isotherm routing, porous-bead accessible area, EDC/NHS chemistry, protein thiol pretreatment, pH-dependent kinetics, and washing/residual reagent tracking. These are real limitations in the current platform and are relevant to wet-lab use.

However, the plan is too broad for a single M2 update and underestimates several architecture dependencies. Some proposed items are scientifically valid but not yet specified tightly enough to be reliable. The highest-risk areas are M3 routing, EDC/NHS spacer chemistry, protein pretreatment state, washing/residual toxic tracking, and the assumption that existing M3 transport code can directly consume SMA or IMAC isotherms.

Primary conclusion: proceed with v5.9 only after splitting it into contract/routing, IMAC, spacer/EDC, protein-state, and washing sub-phases. The plan is directionally sound, but the acceptance criteria must be strengthened to prevent scientifically plausible but operationally unreliable simulations.

## 3. Current System Structure and M2 Role

M2 is the post-fabrication functionalization module. It receives an M1-generated hydrogel microsphere state and transforms it into a chemically functional chromatographic or catalytic medium. Its correct role is to track:

- ACS inventory and conservation.
- Activation, crosslinking, coupling, spacer creation, quenching, hydrolysis, and loss pathways.
- Surface and pore accessibility.
- Functional ligand density, not just nominal coupled ligand density.
- Mechanical changes after secondary crosslinking.
- Chemistry confidence, hazard, validity-window, and calibration metadata.
- A stable `FunctionalMediaContract` for M3 process simulation.

The current implementation is a v5.8-class system:

- 42 reagent profiles are present.
- 6 executable M2 step types exist in backend code, including `SPACER_ARM`.
- 8 ACS site types exist, including `AMINE_DISTAL` and `MALEIMIDE`.
- SM(PEG)n profiles and Cys-protein coupling profiles exist.
- Maleimide decay is partially modeled before protein coupling.
- M2 tests pass for current workflows.

Important current gaps:

- UI still exposes only the older five M2 chemistry classes and does not expose `SPACER_ARM`.
- UI reagent lists do not expose the full v5.8 profile set.
- `FunctionalMediaContract` does not yet include `accessible_area_per_bed_volume`, q_max uncertainty bounds, activity uncertainty, final ligand profile key, or M3 support level.
- M3 does not consume `binding_model_hint` and defaults to Langmuir/competitive Langmuir behavior.
- The single-component LRM transport solver currently expects an isotherm with `equilibrium_loading(C)`; SMA and IMAC isotherms have different signatures and cannot be routed in by a simple class substitution without adapters or solver changes.

## 4. Overall Plan Assessment

The v5.9 plan is scientifically ambitious and mostly points in the correct direction. It correctly identifies that the platform cannot be considered credible for realistic functional media simulation unless M2 outputs actually control M3 process behavior. It also correctly recognizes that IMAC metal loading, protein thiol state, and EDC/NHS carboxyl activation are missing chemistry states.

The main weakness is release-scope control. The plan labels six items as must-have, but these span M2 chemistry, M3 isotherm routing, state-machine expansion, uncertainty propagation, and GMP-style washing. Each of those changes can affect core contracts and tests. Treating them as parallel low-risk work nodes is not reliable.

Recommended framing: v5.9 should be a contract and process-routing release first, then a chemistry-expansion release. Otherwise, new chemistry may produce outputs that M3 still interprets incorrectly.

## 5. Major Findings

### F1. M3 routing is correctly prioritized but technically underestimated

Severity: Critical  
Area: M2-to-M3 simulation validity  

The plan correctly makes `binding_model_hint` routing a v5.9 must-have. Without it, M3 will treat IEX, IMAC, HIC, and affinity outputs too generically. This is currently one of the largest scientific validity gaps in the system.

However, the proposed change is not only a dispatch table. Current `run_breakthrough()` defaults to `LangmuirIsotherm`, and `solve_lrm()` calls `isotherm.equilibrium_loading(Cp)` as a single-argument function. `SMAIsotherm` needs salt concentration. `IMACCompetitionIsotherm` needs protein and imidazole concentrations. Gradient elution has placeholders for salt/pH effects, but those are not wired into the LRM equilibrium calculations.

Recommendation:

Implement isotherm adapters or extend the transport interface before adding routing:

- `EquilibriumAdapter.equilibrium_loading(C, process_state)`
- process-state fields for salt, pH, imidazole, sugar competitor, and gradient value
- explicit single-component versus multicomponent routing
- tests proving that IEX does not call Langmuir and IMAC receives imidazole concentration

Do not consider WN-D2 complete if it only instantiates different isotherm classes.

### F2. `accessible_area_per_bed_volume` is scientifically necessary but must preserve area-basis consistency

Severity: High  
Area: q_max mapping, porous-bead capacity  

The plan correctly identifies that external geometric area alone can misestimate q_max for porous media. The current M2 surface model already computes per-particle external, internal, reagent-accessible, and ligand-accessible areas. The correct bed-volume conversion should be based on particle count per bed volume:

```text
accessible_area_per_bed_volume = accessible_area_per_particle * (1 - eps_bed) / V_particle
```

This is equivalent to scaling external area by the ratio of accessible area to external area, but using the per-particle formula is clearer and less error-prone.

Recommendation:

Add separate fields:

- `reagent_accessible_area_per_bed_volume`
- `ligand_accessible_area_per_bed_volume`
- `capacity_area_basis`

Small-molecule ligands, protein ligands, and porous-particle process capacity should not all use the same area basis by default.

### F3. IMAC metal charging is a real missing state, but it should not live only on `ReagentProfile`

Severity: High  
Area: IMAC chemistry and state tracking  

The plan correctly identifies metal charging as must-have. IDA/NTA without loaded metal is not functional IMAC media.

The proposed equilibrium expression is useful as a first approximation:

```text
metal_loaded_fraction = K_assoc * C_metal / (1 + K_assoc * C_metal)
```

However, metal loading is a state of the functionalized microsphere, not only a static property of the chelator or metal reagent. It must be stored on the material state or on the relevant ACS/ligand profile after the charging step.

Scientific limitations:

- Association constants depend strongly on pH, ionic strength, chelator density, metal identity, and competing ligands.
- Metal hydrolysis/precipitation can occur at inappropriate pH.
- EDTA stripping and metal charging are different workflows and should not be treated as the same reaction type.
- Imidazole is primarily a process competitor in M3, not a complete substitute for EDTA stripping/regeneration logic.

Recommendation:

Add metal state to the functional media:

- `metal_ion`
- `metal_loaded_fraction`
- `metal_capacity_density`
- `metal_charging_pH`
- `metal_state_confidence`
- `stripped_fraction`

Then update FMC from this material state, not from the original IDA/NTA reagent profile.

### F4. EDC/NHS chemistry is scientifically important but under-specified for the current ACS model

Severity: High  
Area: carboxyl activation, AHA spacer path  

The plan says EDC/NHS completes the AHA spacer pathway. That is correct chemically, but implementation is not as simple as adding `NHS_ESTER` and an activation profile.

Current v5.8 backend has `CARBOXYL` as an ACS type, but there is no executable AHA spacer-arm route that creates a carboxyl-terminated distal ACS. AHA currently exists as metadata in earlier phases; v5.8 spacer-arm executable profiles focus on amine-distal and maleimide paths. To support:

```text
EPOXIDE -> AHA carboxyl terminus -> EDC/NHS -> NHS_ESTER -> amine coupling
```

the system needs either:

- `CARBOXYL_DISTAL` as a distinct site type, or
- careful reuse of `CARBOXYL` with a provenance field showing it is a distal spacer site, not native carboxyl chemistry.

Recommendation:

Do not implement EDC/NHS until the AHA spacer-arm source of carboxyl sites is executable and validated. Add acceptance tests for:

- ECH activation -> AHA spacer-arm -> distal carboxyl creation
- EDC/NHS activation -> NHS_ESTER creation
- NHS ester hydrolysis during delay/coupling
- amine ligand/protein coupling from NHS_ESTER

### F5. EDC/NHS kinetics should not be overclaimed from a pseudo-single-step model

Severity: Medium-High  
Area: reaction kinetics  

The plan proposes a pseudo-single-step EDC/NHS activation if a sequential mechanism is too complex. That is acceptable for screening, but it must be labeled as semi-quantitative. Real EDC/NHS chemistry involves O-acylisourea formation, NHS ester stabilization, hydrolysis, possible N-acylurea side products, and a pH switch between carboxyl activation and amine coupling.

Recommendation:

Use confidence labels:

- `ranking_only` for pseudo-single-step EDC/NHS.
- `semi_quantitative` only after hydrolysis and pH-window tests are implemented.
- `calibrated` only after wet-lab activation/coupling yield data are available.

### F6. Protein pretreatment is necessary for maleimide-thiol coupling but introduces a second state machine

Severity: High  
Area: protein chemistry, workflow state  

The plan correctly notes that protein reduction modifies the protein, not the bead. That means `PROTEIN_PRETREATMENT` cannot be handled as a normal ACS-consuming surface chemistry step.

The proposed `protein_state` dict is reasonable as a minimal start, but it is not dependency-free. It requires:

- workflow ordering validation,
- UI support,
- state persistence through the orchestrator,
- reagent-removal checks before maleimide coupling,
- protein identity matching between pretreatment and coupling,
- elapsed-time/reoxidation handling.

Wet-lab issues:

- DTT and beta-mercaptoethanol interfere with maleimide unless removed.
- TCEP is preferred but still requires validated compatibility and removal assumptions.
- Over-reduction can damage some proteins.
- Disulfide reduction may change protein activity or oligomeric state.

Recommendation:

Implement `ProteinPretreatmentState` as a typed dataclass, not a loose dict. Include:

- `protein_key`
- `free_thiol_fraction`
- `activity_retention_after_reduction`
- `excess_reductant_removed`
- `time_since_reduction`
- `reoxidation_rate`
- `pretreatment_warnings`

Block maleimide coupling if residual DTT/free thiol reductant is not removed.

### F7. Washing/residual toxic tracking is useful but should not be a v5.9 must-have unless scoped narrowly

Severity: Medium-High  
Area: compliance, residuals, wet-lab realism  

Washing is valuable, especially for ECH, DVS, glutaraldehyde, EDC/NHS byproducts, and residual metal salts. However, the proposed model is a large new subsystem, not just one step type.

Scientific limitations:

- Soluble residual reagent, pore-retained reagent, adsorbed reagent, and covalently bound reactive groups are different states.
- Regulatory ppm limits are context-specific and must cite the relevant pharmacopeial or GMP basis.
- Convective washing in packed porous beads is not simply diffusion-out with a single pore depth.
- Partitioning into hydrogel phases can dominate for hydrophobic reagents.

Recommendation:

For v5.9, scope washing to a simple advisory residual model:

- track only soluble residual reagent from the previous step,
- report `screening_only`,
- require user-specified wash volumes and flow rate,
- do not claim GMP pass/fail without validated limits and assays.

Full residual toxicological compliance should be v6.0 or later.

### F8. pH-dependent rate scaling is desirable, but pKa-only scaling is incomplete

Severity: Medium  
Area: kinetic realism  

The plan correctly identifies that current pH handling is mostly validity-window checking. A pH scaling factor would improve realism.

However, nucleophile pKa alone is not enough. Reaction rates also depend on:

- electrophile stability,
- hydrolysis rate,
- buffer catalysis,
- protein stability,
- thiol oxidation,
- ligand ionization,
- competing amines or salts.

Recommendation:

Implement pH scaling behind a disabled default, as the plan suggests, but add per-chemistry scaling modes:

- amine nucleophile fraction,
- thiolate fraction,
- NHS hydrolysis scaling,
- maleimide amine-crossreaction warning,
- epoxide/vinyl sulfone alkaline hydrolysis scaling.

Do not state that current rankings are always correct without pH scaling; rankings can change across pH for amine versus thiol versus hydrolysis-dominated chemistries.

### F9. Irreversible affinity binding cannot be implemented by only adding a flag to equilibrium Langmuir

Severity: Medium-High  
Area: M3 dynamics, streptavidin-biotin simulation  

The plan proposes adding `is_irreversible` to `LangmuirIsotherm`. This is not sufficient if the transport solver still relaxes toward an equilibrium loading `q_eq(C)`. Irreversible adsorption requires kinetic handling where desorption is zero or negligible:

```text
dq/dt = k_ads * C * (q_max - q)
```

rather than:

```text
dq/dt = k_ads * (q_eq(C) - q)
```

Recommendation:

Add an irreversible adsorption kinetic model or an isotherm/kinetics adapter. Confirm with a washout test that bound material does not elute when feed concentration drops to zero.

### F10. Target-specific defaults are useful but must be clearly separated from calibrated predictions

Severity: Medium  
Area: M3 usability, scientific trust  

Target defaults improve usability, but they can create false precision. IgG, His-tag proteins, GST fusions, and heparin-binding proteins vary widely in charge, hydrodynamic radius, affinity, and stability.

Recommendation:

Every target default should carry:

- source or rationale,
- confidence tier,
- valid pH/conductivity range,
- whether it is a placeholder or calibrated,
- warning if used for quantitative prediction.

### F11. Deferring PSD and M1 uncertainty is scientifically defensible, but limits v5.9 claims

Severity: Medium  
Area: uncertainty and accessibility  

Deferring pore-size distribution and batch-to-batch variability to v6.0 is reasonable. However, v5.9 should then avoid implying accurate porous-bead capacity prediction. The accessible-area model still uses mean pore size and empirical accessibility. Bimodal or broad pore distributions can strongly affect macromolecule access.

Recommendation:

When `accessible_area_per_bed_volume` is added, include a warning:

```text
Capacity uses mean-pore accessibility. Broad or bimodal pore-size distributions are not represented.
```

### F12. The UI plan is incomplete

Severity: High  
Area: user reliability  

The v5.9 plan adds new step types and workflows, but does not allocate enough UI work. Current UI still does not expose backend `SPACER_ARM`, and it does not expose many v5.8 profiles. v5.9 adds `METAL_CHARGING`, `PROTEIN_PRETREATMENT`, and `WASHING`, which are even harder for users to order correctly.

Recommendation:

Add a required UI work node. The UI must:

- expose `SPACER_ARM` before adding v5.9 step types,
- filter reagents by current available ACS or protein state,
- display intermediate ACS state after each step,
- block metal charging before IDA/NTA coupling,
- block maleimide-thiol coupling before required protein pretreatment or explicit thiol accessibility,
- block DTT-reduced protein coupling unless reductant removal is confirmed,
- display residual reagent warnings after washing.

### F13. Current `FunctionalMediaContract` is too small for the v5.9 scope

Severity: High  
Area: system contract design  

The plan adds fields in several separate items, but the contract should be redesigned coherently. Current FMC lacks the fields needed to make v5.9 reliable.

Recommendation:

Add a consolidated v5.9 FMC extension:

- `accessible_area_per_bed_volume`
- `reagent_accessible_area_per_bed_volume`
- `ligand_accessible_area_per_bed_volume`
- `final_ligand_profile_key`
- `m3_support_level`
- `activity_retention_uncertainty`
- `functional_density_uncertainty`
- `q_max_lower`
- `q_max_upper`
- `metal_ion`
- `metal_loaded_fraction`
- `process_state_requirements`
- `residual_reagent_warnings`

This should be implemented before M3 routing.

### F14. Release scope and effort estimates are optimistic

Severity: Medium-High  
Area: project reliability  

The plan estimates 58 total hours for implementation, testing, documentation, and audit prep. That may be optimistic because the work touches:

- M2 state machine,
- M2 reagent data,
- M2 workflow validation,
- M2 UI,
- M2-to-M3 contract,
- M3 isotherm routing,
- M3 transport solver interfaces,
- M3 tests,
- residual washing state,
- target defaults.

Recommendation:

Split v5.9 into smaller releases:

- v5.9.0: FMC contract extension, accessible area, M3 routing adapters.
- v5.9.1: IMAC metal charging.
- v5.9.2: protein pretreatment and maleimide-thiol completion.
- v5.9.3: EDC/NHS and AHA carboxyl path.
- v5.9.4: washing/residual advisory model.

## 7. Revised v5.9 Acceptance Criteria

The current acceptance criteria should be expanded as follows:

1. M3 routing includes working adapters for Langmuir, SMA, IMAC, and irreversible affinity, not only a dispatch table.
2. M3 routing tests verify that `binding_model_hint="charge_exchange"` invokes SMA with salt state and that `"metal_chelation"` invokes IMAC with imidazole state.
3. `FunctionalMediaContract` carries accessible area per bed volume and explicitly labels the area basis used for q_max.
4. q_max values are reported with confidence and, where available, lower/upper uncertainty bounds.
5. Metal charging state is stored on the functional material or profile state after execution, not only on the original reagent profile.
6. EDTA stripping is tested as a distinct state-changing operation from metal charging.
7. EDC/NHS is blocked until an executable carboxyl-distal or AHA spacer path exists.
8. NHS ester hydrolysis is represented during coupling or delay.
9. Protein pretreatment uses a typed `ProteinPretreatmentState`.
10. Maleimide-thiol coupling reads thiol accessibility from protein state when a pretreatment step is present.
11. UI exposes all backend step types required by v5.9 and blocks invalid ordering.
12. Washing outputs are labeled advisory unless validated regulatory limits and residual assays are provided.
13. Existing v5.8 workflows, including SM(PEG)n and `SPACER_ARM`, continue to pass.
14. M3 mass-balance warnings are explicitly classified in v5.9 reports so users know when process simulation quality is only caution-level.

## 8. Recommended Development Sequence

Recommended sequence:

1. Extend `FunctionalMediaContract` first.
2. Add M3 isotherm adapters and routing tests.
3. Add accessible-area q_max mapping and area-basis reporting.
4. Add UI support for existing `SPACER_ARM` and v5.8 profiles.
5. Add IMAC metal charging and stripping.
6. Add protein pretreatment state and maleimide-thiol integration.
7. Add EDC/NHS only after executable AHA/carboxyl-distal creation is implemented.
8. Add washing as an advisory residual model.
9. Add pH scaling after the above pathways are stable.
10. Add target defaults only after routing can consume them.

This sequence reduces the risk that new chemistry produces outputs that M3 still misuses.

## 9. Final Reliability Assessment

Current system reliability:

- M2 backend reliability is good for implemented v5.8 workflows.
- M2 UI reliability lags backend capability.
- M2-to-M3 contract reliability is structurally useful but incomplete for v5.9.
- M3 process model reliability is acceptable for existing generic Langmuir/competitive simulations, but not yet reliable for mode-specific IEX, IMAC, HIC, lectin, or irreversible affinity routing.

v5.9 plan reliability as written:

- Scientifically directionally correct.
- Too broad and optimistic as a single release.
- Underestimates M3 interface changes.
- Underestimates UI workflow changes.
- Overstates readiness of EDC/NHS and washing as must-have items.

v5.9 plan reliability after correction:

- Strong candidate for a scientifically meaningful next phase.
- Suitable for semi-quantitative wet-lab planning and M2-to-M3 process exploration.
- Still not a calibrated substitute for measured resin characterization, breakthrough curves, or regulatory residual testing.

## 10. Final Recommendation

Proceed with v5.9, but revise the plan before implementation. The must-have core should be narrowed to contract correctness, accessible area, M3 routing, and IMAC state. Protein pretreatment, EDC/NHS, pH scaling, and washing should be added only after their state and validation requirements are made explicit.

The most important correction is architectural: M3 must truly consume M2 output. If M2 produces chemically detailed media but M3 still uses generic Langmuir defaults, the integrated simulator will appear more advanced than it actually is. Fix the contract and routing first, then expand chemistry.
