|---|---|---|
| D7 | Salt-dependent HIC isotherms | User calibration framework |

| D10 | Lectin-specific M3 elution models | Sugar competition constants + calibration |

| D12 | Metal leaching detail | Cycle-dependent lifetime model |

Additionally, v5.8-v5.9 development revealed four **cross-cutting needs** not captured by any single deferred item:

1. **User calibration framework** --- a general mechanism for users to supply measured parameters (K values, q_max, selectivity) that override semi-quantitative defaults. Enables D7, D10, and future calibrated profiles.
2. **M1 uncertainty interface** --- extends M1ExportContract with uncertainty distributions. Enables D14, D15.
3. **Cycle-dependent lifetime model** --- exponential decay + CIP degradation model for long-term performance projection. Enables D12, D16.
4. **Complete UI overhaul** --- the visualization layer must expose all 9 step types, 52+ profiles, calibration inputs, gradient configuration, and lifetime projections.

v6.0 delivers these frameworks and the chemistry they unlock. Items that remain scientifically unready are deferred to v7.0.

## 2. Item-by-Item Assessment

### 2.1 F1: User Calibration Framework (NEW --- enables D7, D10)

**SCIENTIFIC ADVISOR:**
- Readiness: HIGH. The concept is straightforward: allow users to supply measured isotherm parameters (q_max, K_eq, selectivity, salt sensitivity coefficients) that override the semi-quantitative defaults generated by M2. This is standard practice in chromatography modeling software (e.g., Chromeleon, CADET).
- Data requirements: None beyond a JSON/YAML schema for calibration data. The framework itself is data-agnostic.
- Priority: **MUST-HAVE.** This is the single highest-leverage item in v6.0. Without it, every profile with `m3_support_level="requires_user_calibration"` (Protein L, Con A, WGA, all lectin profiles) and every HIC profile remains a dead end in M3.

**ARCHITECT:**
- New module: `src/emulsim/calibration/` with:
  - `calibration_data.py`: `CalibrationEntry` dataclass (profile_key, parameter_name, measured_value, units, temperature, pH, salt_type, salt_concentration, confidence, source_reference).
  - `calibration_store.py`: `CalibrationStore` class --- load/save/query calibration datasets. In-memory dict backed by JSON file.
  - `calibration_applier.py`: `apply_calibration(fmc, calibration_store)` --- overrides FMC defaults with measured values before M3 consumption.
- Modify `module3_performance/orchestrator.py`: accept optional `CalibrationStore`, apply before isotherm instantiation.
- Modify FMC: add `calibration_overrides: dict[str, float]` field for transparency.
- UI: calibration import panel (CSV/JSON upload or manual entry).
- LOC estimate: ~450 (framework) + ~120 (UI) + ~80 (tests) = **~650**

**DEV-ORCHESTRATOR:**
- Work node: **WN-F1**
- Model tier: Opus (new subsystem design)
- Dependencies: None. Foundation for D7, D10.

### 2.2 F2: M1 Uncertainty Interface (NEW --- enables D14, D15)

**SCIENTIFIC ADVISOR:**
- Readiness: MEDIUM. Requires coordinated changes to M1ExportContract (external to M2/M3). The statistical framework is standard: either analytical error propagation (fast, linear approximation) or Monte Carlo sampling (accurate, 100x slower).
- Recommended approach: Add CV (coefficient of variation) fields to M1ExportContract for bead_d50, porosity, pore_size_mean, nh2_bulk_concentration, oh_bulk_concentration. M2 orchestrator wraps `run()` in a Monte Carlo loop (N=100-500) and reports percentile bounds.
- Data requirements: M1 must provide uncertainty estimates. If M1 cannot provide them, use literature-typical CVs as defaults (CV_d50 = 0.05-0.15, CV_porosity = 0.05, CV_pore_size = 0.10).
- Priority: **MUST-HAVE.** Without uncertainty propagation, M2/M3 outputs present false precision. A q_max of "12.3 mg/mL" with no error bar is misleading when the bead d50 has 10% CV.

**ARCHITECT:**
- New dataclass: `M1UncertaintyContract` with CV fields for all M1 outputs.
- Extend `M1ExportContract` (in `datatypes.py`) with `uncertainty: Optional[M1UncertaintyContract]`.
- New module: `src/emulsim/uncertainty/` with:
  - `monte_carlo.py`: `run_monte_carlo(orchestrator, m1_export, n_samples, seed)` -> `UncertaintyResult` (percentiles on all FMC fields).
  - `propagation.py`: analytical first-order error propagation as fast alternative.
- Modify FMC: add `q_max_p05`, `q_max_p95`, `functional_density_p05`, `functional_density_p95` fields.
- LOC estimate: ~350 (framework) + ~100 (M1 contract extension) + ~80 (tests) = **~530**

**DEV-ORCHESTRATOR:**
- Work node: **WN-F2**
- Model tier: Opus (Monte Carlo wrapper + contract negotiation)
- Dependencies: M1 team must agree on uncertainty contract fields. Can prototype with default CVs.

### 2.3 F3: Cycle-Dependent Lifetime Model (NEW --- enables D12, D16)

**SCIENTIFIC ADVISOR:**
- Readiness: MEDIUM-HIGH. The model is empirically well-characterized for commercial resins:
  - Protein A: activity decays ~1-3% per CIP cycle (0.1 M NaOH). Typical lifetime: 100-300 cycles.
  - IMAC Ni-NTA: metal leaching ~0.1-1% per cycle. Stripped capacity decays linearly with cycle count.
  - Covalent ligands (IEX, HIC): very stable (>1000 cycles). Degradation dominated by fouling, not bond cleavage.
- Model: `activity(n) = activity_0 * exp(-k_deact * n) * (1 - f_fouled(n))` where `k_deact` is chemistry-specific and `f_fouled` is a simple linear accumulator with CIP partial recovery.
- Data requirements: k_deact per chemistry class (literature values available for Protein A, IMAC, IEX). Fouling model is user-calibratable.
- Priority: **MUST-HAVE.** Process economics and column lifetime are the primary industrial concerns. A simulator that only models fresh-resin performance has limited practical value.

**ARCHITECT:**
- New module: `src/emulsim/lifetime/` with:
  - `lifetime_model.py`: `LifetimeModel` class with `predict_performance(fmc, n_cycles, cip_conditions)` -> `LifetimeProjection`.
  - `LifetimeProjection` dataclass: arrays of (cycle_number, effective_q_max, effective_activity, metal_remaining, cumulative_leachate).
  - `degradation_rates.py`: per-chemistry k_deact constants with literature references.
- Add `k_deactivation_per_cycle: float` and `cip_sensitivity: float` fields to `ReagentProfile`.
- Add lifetime plotting to visualization.
- LOC estimate: ~300 (model) + ~80 (profile updates) + ~60 (UI) + ~60 (tests) = **~500**

**DEV-ORCHESTRATOR:**
- Work node: **WN-F3**
- Model tier: Opus (new subsystem with process economics implications)
- Dependencies: None. Self-contained.

### 2.4 D7: Salt-Dependent HIC Isotherms

**SCIENTIFIC ADVISOR:**
- Readiness: HIGH (now that F1 provides calibration framework). The HIC isotherm model is:
  `q = q_max * K_eff(C_salt) * C / (1 + K_eff(C_salt) * C)`
  where `K_eff = K_0 * exp(m * C_salt)` and `m` is the molal surface tension increment.
- Literature values for m: butyl ~2.5 (mol/kg)^-1, octyl ~4.0 (mol/kg)^-1, phenyl ~1.8 (mol/kg)^-1 with ammonium sulfate. These are protein-dependent; F1 calibration allows user override.
- Data requirements: m_salt per ligand type (available from Melander & Horvath, 1977; Queiroz et al., 2001). Default values sufficient for ranking.
- Priority: **MUST-HAVE.** HIC is the third major chromatography mode after IEX and affinity. Butyl and Octyl profiles already exist (profiles 28, 33) with `binding_model_hint="salt_promoted"` but M3 uses plain Langmuir, which does not capture the defining characteristic of HIC --- salt-dependent binding.

**ARCHITECT:**
- New file: `src/emulsim/module3_performance/isotherms/hic.py` with `HICIsotherm` class.
- Fields: `q_max`, `K_0`, `m_salt`, `salt_type` (default "ammonium_sulfate").
- Extends EquilibriumAdapter: process_state must include `salt_concentration`.
- Update M3 routing: `binding_model_hint="salt_promoted"` -> `HICIsotherm`.
- Add `m_salt_default` field to butyl/octyl/phenyl ReagentProfiles.
- CalibrationStore can override K_0, m_salt per user data.
- LOC estimate: ~180 (isotherm) + ~40 (routing) + ~40 (profiles) + ~60 (tests) = **~320**

**DEV-ORCHESTRATOR:**
- Work node: **WN-D7**
- Model tier: Sonnet (well-characterized isotherm, follows existing patterns)
- Dependencies: **WN-F1** (calibration framework for user overrides)

### 2.5 D10: Lectin-Specific M3 Elution Models

**SCIENTIFIC ADVISOR:**
- Readiness: MEDIUM-HIGH (with F1 calibration). The competitive binding model is identical in form to IMAC imidazole competition:
  `q_protein = q_max * K_p * C_p / (1 + K_p * C_p + K_sugar * C_sugar)`
  where K_sugar is the lectin-sugar association constant.
- Literature values: K_mannose-ConA ~ 8.2e3 M^-1 (Mandal et al., 1994), K_GlcNAc-WGA ~ 5e2 M^-1 (Bains et al., 1992). These vary 2-5x with Ca2+/Mn2+ cofactor concentration.
- Requires: Generalize IMACCompetitionIsotherm to `CompetitiveAffinityIsotherm` accepting arbitrary competitor species.
- Priority: **NICE-TO-HAVE.** Con A and WGA profiles exist but lectin chromatography is niche compared to Protein A/IEX/HIC. The Langmuir placeholder is adequate for capacity ranking.

**ARCHITECT:**
- Refactor `isotherms/imac.py` -> extract `CompetitiveAffinityIsotherm` base class.
- `IMACCompetitionIsotherm` and new `LectinCompetitionIsotherm` both inherit from it.
- Add `sugar_competitor_K` and `cofactor_requirements` fields to lectin ReagentProfiles.
- EquilibriumAdapter: process_state accepts `competitor_concentration` (generic).
- Routing: `binding_model_hint="lectin_*"` -> `LectinCompetitionIsotherm`.
- CalibrationStore provides measured K_sugar values.
- LOC estimate: ~150 (refactor) + ~60 (lectin isotherm) + ~40 (profiles) + ~60 (tests) = **~310**

**DEV-ORCHESTRATOR:**
- Work node: **WN-D10**
- Model tier: Sonnet (refactor of existing pattern)
- Dependencies: **WN-F1** (calibration for K_sugar)

### 2.6 D9: TMAE via DVS (VS-Amine Coupling Path)

**SCIENTIFIC ADVISOR:**
- Readiness: LOW-MEDIUM. The chemistry is DVS activation of hydroxyl groups to vinyl sulfone, followed by nucleophilic addition of trimethylaminoethanol (TMAE) to form a thioether-linked quaternary amine.
- The problem remains: kinetic data for VS + tertiary amine is scarce. VS-primary amine (k ~ 0.1-1 M^-1s^-1 at pH 9) is well-characterized, but tertiary amines are ~100x less nucleophilic. An estimate of k_VS-TMAE ~ 1e-3 to 1e-2 M^-1s^-1 is plausible but unvalidated.
- Alternative approach: Treat TMAE coupling as a semi-quantitative profile with `confidence="ranking_only"` and estimated kinetics, similar to how Q was handled via glycidyltrimethylammonium chloride on epoxide.
- Priority: **NICE-TO-HAVE.** Q (quaternary amine via epoxide) already covers the strong anion exchange use case. TMAE via DVS is an alternative manufacturing route, not a new capability.

**ARCHITECT:**
- New reagent profile: `tmae_vs_coupling` (reaction_type="coupling", target=VINYL_SULFONE, product=TMAE_LIGAND).
- New ACSSiteType: `TMAE_LIGAND` (or reuse a generic `QUATERNARY_AMINE` type).
- Extend coupling solver to accept VINYL_SULFONE targets (currently only EPOXIDE is a coupling target).
- VS target support: generalize `_solve_ligand_coupling_step()` to dispatch on target site type.
- LOC estimate: ~60 (profile) + ~80 (VS target generalization) + ~40 (tests) = **~180**

**DEV-ORCHESTRATOR:**
- Work node: **WN-D9**
- Model tier: Sonnet
- Dependencies: None for implementation. Scientific validation requires literature search or experimental data.

### 2.8 D15: Pore-Size Distribution

**SCIENTIFIC ADVISOR:**
- Readiness: MEDIUM (with F2). PSD integration requires:
  1. M1 provides PSD as log-normal parameters (mu, sigma) or discrete bins.
  2. AccessibleSurfaceModel integrates Renkin exclusion factor over PSD bins.
  3. Result: weighted average f_accessible that captures bimodal effects.
- For unimodal distributions (CV < 0.5), the current mean-pore approach error is < 15%. PSD integration matters primarily for macroporous or bimodal resins.
- Priority: **NICE-TO-HAVE.** The mean-pore approximation is adequate for most agarose-based media. PSD matters for controlled-pore glass and silica, which are not current EmulSim targets.

**ARCHITECT:**
- Extend `M1ExportContract`: add `pore_size_distribution: Optional[tuple[list[float], list[float]]]` (diameters_nm, volume_fractions).
- Extend `AccessibleSurfaceModel.compute()`: if PSD provided, integrate f_accessible over bins.
- LOC estimate: ~120 (PSD integration) + ~40 (M1 contract) + ~40 (tests) = **~200**

**DEV-ORCHESTRATOR:**
- Work node: **WN-D15**
- Model tier: Sonnet
- Dependencies: **WN-F2** (shares M1 contract extension)

### 2.9 D16: Protein Leaching/Deactivation

**SCIENTIFIC ADVISOR:**
- Readiness: HIGH (with F3 lifetime model). The exponential decay model is straightforward and well-validated for Protein A resins (published data from Cytiva MabSelect, Repligen AMSPHERE).
- Key parameters per chemistry class:
  - Protein A (random amine coupling): k_deact ~ 0.005-0.02 per NaOH CIP cycle

### 2.10 D12: Metal Leaching Detail (Cycle-Dependent)

**SCIENTIFIC ADVISOR:**
- Readiness: HIGH (with F3). Metal leaching is a special case of the lifetime model where the degradation mode is loss of the metal ion rather than loss of ligand activity. The model:
  `metal_remaining(n) = metal_loaded_fraction * (1 - f_leach)^n`
  where f_leach ~ 0.005-0.01 per cycle for Ni-NTA with standard CIP.
- Priority: **MUST-HAVE.** Directly delivered by F3 with IMAC-specific leaching parameters.

**ARCHITECT:**
- Covered by WN-F3. The LifetimeModel must track `metal_remaining` as a separate output alongside `effective_activity`.
- Additional LOC beyond F3: ~40 (IMAC-specific leaching output, metal leachate concentration in eluate).

### 2.12 F5: Multi-Component Gradient Elution (Salt/pH/Imidazole)

**SCIENTIFIC ADVISOR:**
- Readiness: MEDIUM-HIGH. The gradient module (`gradient.py`, 245 lines) currently supports linear salt gradients. Multi-component gradients are needed for:
  - IEX: salt gradient (already supported)
  - IMAC: imidazole step gradient (5 mM -> 20 mM -> 250 mM -> 500 mM)
  - Lectin: sugar gradient
  - HIC: reverse salt gradient (high salt -> low salt)
  - Mixed mode: combined pH + salt gradient
- The LRM transport solver needs to accept time-varying process_state, not just a fixed state. EquilibriumAdapter must re-evaluate at each time step.
- Priority: **MUST-HAVE.** IMAC elution with imidazole gradient is the #1 use case after IEX salt gradient. Without it, IMAC breakthrough is modeled but elution is not.

**ARCHITECT:**
- Extend `gradient.py`: `GradientProgram` class with multiple components (salt, pH, imidazole, sugar). Each component has a piecewise-linear profile: `[(t0, c0), (t1, c1), ...]`.
- Modify `transport/lumped_rate.py`: `solve_lrm()` accepts `gradient_program` and updates `process_state` at each time step.
- EquilibriumAdapter: `update_state(process_state)` method called per time step.
- Add step-gradient convenience functions for IMAC (imidazole steps) and HIC (reverse salt).
- LOC estimate: ~200 (gradient program) + ~120 (LRM integration) + ~60 (adapter update) + ~80 (tests) = **~460**

**DEV-ORCHESTRATOR:**
- Work node: **WN-F5**
- Model tier: Opus (transport solver modification is high-risk)
- Dependencies: **WN-D7** (HIC needs reverse gradient), **WN-F1** (calibration for gradient optimization)

## 8. What Constitutes "v6.0 Complete"

v6.0 is shippable when ALL of the following acceptance criteria are met:

### Framework Criteria

1. **Calibration framework operational.** A user can load a JSON calibration file, and M3 chromatogram changes to reflect measured parameters instead of semi-quantitative defaults.
2. **Uncertainty propagation functional.** Given M1 bead_d50 CV=0.10, Monte Carlo (N=100) produces q_max confidence intervals on FMC. P5/P95 bounds are populated.
3. **Lifetime model validated.** Given Protein A resin with k_deact=0.01, lifetime projection over 200 cycles shows monotonic activity decay to ~13.5% of initial. Metal leaching for IMAC shows linear Ni2+ loss.

### Chemistry Criteria

4. **HIC salt-dependent isotherm works.** Butyl profile with ammonium sulfate gradient produces characteristic HIC elution: bind at high salt, elute at low salt. K_eff varies >10x over 0-2M (NH4)2SO4 range.
5. **IMAC imidazole gradient elution works.** His-tag protein binds at 5 mM imidazole, elutes at 250 mM imidazole. Step gradient produces sharp elution peak.
6. **Batch variability propagates.** FMC q_max with and without uncertainty shows meaningful confidence intervals (P95/P5 ratio > 1.3 for CV_d50=0.10).

### UI Criteria

7. **All 9 step types accessible in UI.** No hidden functionality.
8. **All profiles visible and selectable.** Dynamic filtering by step type.
9. **Gradient builder functional.** User can define multi-component gradient (at minimum: linear salt and step imidazole).

### Regression Criteria

10. **All existing v5.9 tests pass.** Zero regressions.
11. **Conservation invariants hold** for all new and existing workflows.
12. **Profile count >= 55.** All new profiles have non-zero parameters with literature citations.

## 9. What is v7.0 Scope

| Item | Description | Reason for Deferral |
|---|---|---|
| D9 | TMAE via DVS | Kinetic data still not available; Q covers the use case adequately |
| Mechanistic pH-T coupling | Arrhenius-type temperature and pH combined kinetic models | Requires thermodynamic database; sigmoid scaling is sufficient for ranking |
| Gradient optimization | Multi-objective optimization of gradient shape for resolution/yield/purity | Requires optimizer framework (scipy.optimize integration or Bayesian); manual gradients are sufficient |
| GMP washing validation | Quantitative washing model with regulatory compliance database | Requires regulatory limit database (EP, USP); advisory model is sufficient |
| PDB-based orientation | Predict ligand orientation and activity retention from protein crystal structure | Requires PDB parser and surface accessibility calculation; empirical values sufficient |
| Multi-column continuous chromatography | Simulated moving bed (SMB) and periodic counter-current (PCC) models | Major transport solver extension; single-column batch is sufficient for media design |
| Temperature-dependent isotherms | Van't Hoff analysis for isotherm temperature dependence | Requires enthalpy data; room temperature assumption is adequate |
| Fouling model | Irreversible fouling accumulation reducing effective capacity over cycles | Requires fouling kinetics per protein/resin pair; lifetime model captures net effect |
